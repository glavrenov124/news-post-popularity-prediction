# News Post Popularity Prediction

Сервис для прогнозирования популярности новостных постов на основе текста и изображения. Проект позволяет оценить ожидаемое количество просмотров, уровень вовлечённости, соответствие текста и изображения, а также сформировать рекомендации по улучшению публикации.

## Возможности сервиса

Проект включает backend на FastAPI и frontend на Streamlit. Пользователь может:

- ввести текст поста;
- выбрать домен публикации;
- указать дату и время размещения;
- прикрепить одно или несколько изображений;
- получить прогноз просмотров;
- получить прогноз engagement rate;
- оценить соответствие текста и изображения;
- получить рекомендации по улучшению текста;
- получить улучшенную версию текста с помощью LLM, если настроены соответствующие переменные окружения.

## Архитектура проекта

Проект состоит из нескольких частей:

- **Frontend (Streamlit)** — пользовательский интерфейс для ввода параметров публикации и отображения результатов;
- **Backend (FastAPI)** — API для обработки запросов, вызова моделей, расчёта рекомендаций и улучшения текста;
- **ML-часть** — подготовка данных, извлечение признаков, построение эмбеддингов, обучение моделей и логирование;
- **Utils** — вспомогательные функции для работы с датами, DataFrame и правилами рекомендаций.

## Основные API-методы

| Метод | Назначение |
|---|---|
| `/ping` | Проверка доступности сервиса |
| `/predict/views` | Прогноз количества просмотров |
| `/predict/engagement-rate` | Прогноз уровня вовлечённости |
| `/analyze/clip-score` | Оценка соответствия текста и изображения |
| `/recommend/text` | Формирование рекомендаций по улучшению текста поста на основе признаков публикации и статистик исторических данных |
| `/improve/text` | Улучшение текста поста с помощью LLM |

## Структура проекта

```text
src/
  backend/     # FastAPI backend
  frontend/    # Streamlit frontend
  ml/          # подготовка данных, эмбеддинги, обучение моделей, логирование  моделей

artifacts/ (папка появляется после предобработки данных)
  models/      # обученные модели

```
##Запуск проекта

## 1. Клонирование репозитория
```bash
git clone https://github.com/glavrenov124/news-post-popularity-prediction
cd news-post-popularity-prediction
```
## 2. Установка зависимостей
```bash
poetry install --no-root
```
## 3. Загрузка датасет
```bash
poetry run python scripts/download_data.py
```
## 4. Подготовка данных и эмбеддингов текста
```bash
poetry run python -m src.ml.main prepare-data
poetry run python -m src.ml.main prepare-embeddings
```
## 5. Обучение моделей

```bash
poetry run python -m src.ml.main train --target views
poetry run python -m src.ml.main train --target engagement_rate
```
## 6. Создание файла .env
```text
Для использования функции автоматического улучшения текста через LLM необходимо создать файл .env и указать:

YANDEX_CLOUD_FOLDER=...
YANDEX_CLOUD_API_KEY=...
YANDEX_CLOUD_MODEL="deepseek-v32/latest"

Если переменные не заданы, функция улучшения не будет работать.
```
